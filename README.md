## 🌌 What is ContriFlow?

Most open-source repositories **silently lose contributors** — not because the code is bad, but because basic documentation is missing. No `CONTRIBUTING.md`. No issue templates. No Code of Conduct. New developers land, get confused, and leave.

**ContriFlow** is an AI-powered GitHub repository analyzer that:
- Scans any repo in **under 8 seconds**
- Runs **10 weighted contributor-readiness checks**
- Gives you a **score out of 100** with a grade (A–F)
- Uses **Groq Llama 3.3 AI** to generate ready-to-commit fixes
- Shows **deep analytics** — heatmaps, language charts, commit activity, top contributors

## 🏆 The 10 Checks

<div align="center">

| # | Check | Points | What It Looks For |
|---|-------|--------|-------------------|
| 1 | 📄 README Quality | 25 pts | Length, install section, usage, badges, screenshots |
| 2 | 💬 Repository Description | 8 pts | Meaningful description set on GitHub |
| 3 | 🤝 CONTRIBUTING Guide | 15 pts | CONTRIBUTING.md with onboarding instructions |
| 4 | ⚖️ Open Source License | 8 pts | A recognized LICENSE file |
| 5 | 🛡️ Code of Conduct | 8 pts | CODE_OF_CONDUCT.md for community safety |
| 6 | 🐛 Issue Templates | 8 pts | .github/ISSUE_TEMPLATE/ with bug/feature templates |
| 7 | ⚙️ CI/CD Pipeline | 10 pts | GitHub Actions workflows for automated testing |
| 8 | 📝 Changelog | 5 pts | CHANGELOG.md tracking version history |
| 9 | 🔐 Security Policy | 5 pts | SECURITY.md with vulnerability reporting process |
| 10 | 🟢 Recently Active | 8 pts | Last commit within 6 months |
| | **Total** | **100 pts** | **Grade A → F** |

</div>

## 🚀 Getting Started

### Prerequisites

- Node.js v18+
- GitHub Personal Access Token ([get one here](https://github.com/settings/tokens))
- Groq API Key ([free at console.groq.com](https://console.groq.com))

### Installation

```bash
# 1. Clone the repository
git clone https://github.com/kritikatripathi17/ContriFlow.git
cd ContriFlow

# 2. Install dependencies
npm install

# 3. Create environment file
cp .env.example .env.local
```

### Environment Variables

Create a `.env.local` file in the root:

```env
# GitHub API — for 5,000 req/hr (vs 60 unauthenticated)
GITHUB_TOKEN=ghp_your_token_here

# Groq API — free at console.groq.com
GROQ_API_KEY=gsk_your_key_here
```

### Run Development Server

```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000) and paste any GitHub repo URL!

## 🔌 API Reference

### `POST /api/analyze`

Analyzes a GitHub repository across all 10 checks.

```json
// Request
{
  "repoUrl": "https://github.com/facebook/react"
}

// Response
{
  "success": true,
  "report": {
    "repoData": { "fullName": "facebook/react", "stars": 245810, ... },
    "checks": [ { "id": "readme_quality", "score": 25, "maxScore": 25, ... } ],
    "totalScore": 100,
    "maxScore": 100,
    "grade": "A",
    "languages": [ { "name": "JavaScript", "pct": 49 }, ... ],
    "commitActivity": [ { "week": 1749945600, "total": 33 }, ... ],
    "contributors": [ { "login": "gaearon", "contributions": 1682, ... } ],
    "avgResolutionDays": 7,
    "heatmap": [ { "week": 1749945600, "count": 33 }, ... ]
  }
}
```

### `POST /api/fix`

Generates an AI fix for a failed check using Groq Llama 3.3.

```json
// Request
{
  "checkId": "contributing",
  "checkLabel": "CONTRIBUTING guide",
  "repoName": "facebook/react",
  "detail": "No CONTRIBUTING.md found."
}

// Response
{
  "success": true,
  "fix": "# Contributing to react\n\nWe love your input!..."
}
```

### `POST /api/compare`

Compares two repositories and returns both full reports.

```json
// Request
{
  "repoUrl1": "https://github.com/facebook/react",
  "repoUrl2": "https://github.com/vuejs/vue"
}

// Response
{
  "success": true,
  "report1": { ... },
  "report2": { ... }
}
```

## 🎨 UI Animations

Every animation in ContriFlow is purposeful, not decorative:

```
Homepage
  ├── Neural Network Canvas — 55 particles, mouse repulsion, pulsing glows
  ├── Typing Animation — cycles 4 phrases with realistic typing/deleting
  └── 3D Tilt Cards — Framer Motion useMotionValue for perspective tilt

Report Page
  ├── Grade Ring — SVG draws itself with CSS transition, glows in grade color
  ├── Score Counter — counts 0 → final using easeOutCubic
  ├── Staggered Cards — each slides in from left, 60ms delay between cards
  ├── Confetti Burst — 80 particles rain down on Grade A 🎉
  ├── Progress Bars — animate to correct width on scroll into view
  └── Expandable Cards — smooth height animation on open/close

Analytics
  ├── Heatmap — squares render with hover tooltips
  ├── Language Donut — segments fade in one by one
  ├── Commit Bars — animate up from zero on scroll
  └── Contributor Bars — purple bars animate to contribution width
```

## 💡 The Story Behind ContriFlow

This project was born from a real frustration.

When I participated in my first open source program — **GirlScript Summer of Code** — I was excited to contribute. I found a repo I wanted to work on, but spent hours trying to figure out *how to even start*. No CONTRIBUTING guide, no issue templates, nothing. I gave up.

Then **ELMSoC (Edulink Summer of Code)** — same story. And I thought: *this can't just be me.*

So I built ContriFlow — a tool that tells maintainers, instantly, whether their repo is actually ready for contributors. Because **no contributor should ever feel lost just trying to start.**
